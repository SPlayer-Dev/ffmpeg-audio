# ffmpeg-audio-sys

A Rust crate providing low-level bindings to FFmpeg's C audio libraries.

## Prerequisites

The following tools are required to build and update the FFmpeg bundle:

* [Deno](https://deno.land/) — Runs build scripts (`generate_config.ts`, `extract_slim.ts`)
* [MSYS2](https://www.msys2.org/) (Windows only) — Provides `make`, `git`, and other Unix tools
* Cross-compilation toolchains (for non-host targets, e.g., `gcc-aarch64-linux-gnu` for Linux ARM64)

## Vendor Directory

The `vendor/` directory contains pre-built FFmpeg artifacts:

* **`configs.zip`** — Per-platform configuration files (`config.h`, `config_components.h`, `make_dryrun.log`)
* **`ffmpeg_slim.zip`** — A lightweight FFmpeg source tree with only the required C source and header files

These files are automatically generated by the `Update Bundled FFmpeg` workflow.

### How It Works

1. **Configuration:** GitHub Actions run the FFmpeg `configure` script across various platforms, generating configuration files such as `config.h` and `config_components.h`.
2. **Source Code Extraction:** GitHub Actions execute `make V=1 -n` on each platform, producing a `make_dryrun.log` file. All required C source files are extracted from this log.
3. **Packaging:** The `configs.zip` archive and `ffmpeg_slim` (a lightweight FFmpeg version with only the extracted C source and header files) are retrieved. Both files are small enough to bundle directly with the crate.
4. **Compilation and Binding:** The [cc](https://github.com/rust-lang/cc-rs) crate compiles the C source code, while [bindgen](https://github.com/rust-lang/rust-bindgen) generates the Rust bindings.

### Scripts

The `scripts/` directory contains two Deno scripts:

* **`generate_config.ts`** — Runs FFmpeg's `configure` script for a target platform and extracts the generated configuration files
* **`extract_slim.ts`** — Parses `make_dryrun.log` to extract required C source files and packages them into `ffmpeg_slim.zip`

### Updating Configurations and the FFmpeg Bundle

Run the `Update Bundled FFmpeg` workflow to automatically update `configs.zip` and `ffmpeg_slim` in the `crates/ffmpeg_audio_sys/vendor` directory.

### Supporting New Build Targets

1. Add a build step for the target in the workflow configuration.
2. Modify the scripts in the `scripts` folder to support the new target platform.
